School Auto Chat — User Manual (DEMO)

This is a demo version. Run locally using run.bat.

Overview

Standard chat with an integrated TeacherBot that searches local files inside the knowledge_base folder (TXT/HTML). The bot returns the context of the first occurrence of the search term in each file. Recommendation: keep each term present only once in the files to get predictable results.

Run (Windows)

run.bat

When Windows asks to allow access through the firewall, choose Yes to open the port for local network clients.

Examples (from your knowledge_base)

1. IT (Information Technology) — Field of technology related to the use of computers, networks, software, and data.
35. Firewall — System that filters network traffic.
38. IP Address (Internet Protocol Address) — Logical address assigned to a network device.

These snippets are pulled directly from the files in knowledge_base and show how terms are written for best search results.

Search behavior

  • Search is case-insensitive substring match.
  • Only the first occurrence in a file is returned (context ~300 chars).
  • If no exact match, the bot suggests similar indexed words.

Formatting recommendations

  • Save files in UTF-8 encoding.
  • Write terms explicitly and once per file (e.g., Firewall — System that filters network traffic.).
  • Use sentences ending with periods to allow clean sentence-based context extraction.
  • HTML files: use simple tags (<p>, <h1>) — the bot extracts visible text.
